# ADR-0004 — Контент-как-код с идемпотентным сидером

Статус: принято · Дата: 2026-06-18

## Контекст
У курсов, тестов и задач тренажёра есть авторский контент. Нужно решить, откуда он
берётся на MVP, не загоняя в угол будущую админку.

## Решение
**БД — источник правды в рантайме** (единая модель, прогресс ссылается по ID, фронт
читает из API). На MVP контент **авторится файлами** в `/content`:
- уроки — **Markdown** (`courses/*.md`),
- тесты и задачи тренажёра — **YAML** (`quizzes/*.yaml`, `trainer/*.yaml`),
- пороги уровней — **YAML** (`levels.yaml`).

**Идемпотентный сидер** грузит файлы в БД по стабильным `slug` (повторный запуск
обновляет, а не дублирует). Админка/CMS — вне MVP, но модель и репозитории
спроектированы так, чтобы CRUD добавился позже без переделки.

## Последствия
- (+) Контент версионируется в git, ревью через PR, быстрый старт.
- (+) Рантайм-модель единая, не зависит от способа авторинга.
- (−) Нетехнические авторы пока не могут редактировать без PR (до появления админки).

## Альтернативы
- Сразу админка/CMS — отвергнуто: отдельный продукт внутри продукта, дорого для MVP.
- Файлы как источник правды в рантайме — отвергнуто: мешает прогрессу ссылаться по ID
  и будущей админке.
