# ADR-0007 — Аутентификация: свой тонкий JWT-auth

Статус: принято · Дата: 2026-06-18

## Контекст
Нужны регистрация/вход для SPA (React) + FastAPI. Важны контроль над модульной
структурой и разумная безопасность без избыточной плумбинг-работы на MVP.

## Решение
Своя тонкая реализация в модуле **`users`** (без `fastapi-users`):
- **JWT**: короткоживущий **access-токен в памяти** фронта (не localStorage) +
  **refresh-токен в httpOnly Secure SameSite cookie** с ротацией.
- Пароли — `passlib`/bcrypt.
- Регистрация: **email + пароль**, без email-верификации и OAuth (заложено на будущее).
- Поле **`role`** (`user|admin`) заведено сразу — под будущую админку.
- **Сброс пароля — вне MVP** (нет почтовой инфраструктуры).

## Последствия
- (+) Полный контроль над модульной структурой, без навязанных моделей.
- (+) access не в localStorage → меньше XSS-риск; refresh в httpOnly → недоступен JS.
- (−) Security-чувствительный код пишем сами (малый, протоптанный объём).
- (−) Забытый пароль на MVP = создать новый аккаунт.

## Альтернативы
- `fastapi-users` — быстрее по фичам, но навязывает модели/структуру; отклонён.
- Серверные сессии на cookie — проще, но менее SPA-дружелюбно/масштабируемо.
