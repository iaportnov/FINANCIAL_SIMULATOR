# ADR-0006 — Движок тренажёра за адаптером (Fortune-sheet)

Статус: принято · Дата: 2026-06-18

## Контекст
Нужна Excel-подобная таблица в браузере с поддержкой формул (включая финансовые) для
тренажёра. Выбор движка имеет лицензионные и эксплуатационные риски.

## Решение
**Движок прячем за собственным адаптером** `<SpreadsheetTrainer task onSubmit />` со
стабильным интерфейсом; конкретная библиотека живёт внутри адаптера. Задача
описывается **нейтральной YAML-схемой**, грейдинг — против **нашей** модели ячеек.

На MVP берём **Fortune-sheet** (MIT). Осознанные риски и митигации:
- Поддержка проекта затухает (форк заброшенного Luckysheet; авторы Luckysheet советуют
  Univer). Принято ради скорости старта; адаптер делает переезд умеренным.
- `PMT` в Fortune-sheet не подтверждён. Митигация: при сборке проверить поддержку
  регистрации своих функций → добавить `PMT/IPMT/PPMT` из `@formulajs/formulajs`
  (MIT); если нельзя — авторить финансовые задачи на базовой арифметике (аннуитет из
  первых принципов).

Правила, удерживающие стоимость будущего переезда низкой:
- нейтральная YAML-схема задач (не родной JSON движка);
- грейдинг против нашей модели ячеек;
- толерантный парсинг `must_use_function` (регистр/разделители);
- только ячейки + формулы на MVP (без диаграмм/условного форматирования);
- smoke-набор формул с известными результатами для проверки паритета при смене движка.

## Последствия
- (+) Быстрый старт, чистая MIT-лицензия, переезд на Univer позже — переписать один
  адаптер + перепроверить формулы задач.
- (−) Строим на затухающей ветке; стоимость переезда растёт с числом задач.

## Альтернативы
- **Univer** (Apache-2.0, живой, официальный преемник) — рекомендовался; отклонён в
  пользу более лёгкой интеграции Fortune-sheet. Кандидат №1 при будущей смене.
- Handsontable — платно. HyperFormula — GPL/платно. Своя сетка — много UX-работы.
